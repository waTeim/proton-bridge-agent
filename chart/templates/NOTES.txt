Proton Mail Bridge has been deployed.

=== First-Time Setup (required before the bridge will function) ===

1. Get the pod name:

   export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} \
     -l "app.kubernetes.io/name={{ include "proton-bridge.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" \
     -o jsonpath="{.items[0].metadata.name}")

2. Open an interactive shell in the pod:

   kubectl exec --namespace {{ .Release.Namespace }} -it $POD_NAME -- /bin/sh

3. Inside the pod, start the bridge CLI:

   proton-bridge --cli

4. Log in to your Proton account and verify the connection:

   >>> login
   (follow the prompts to authenticate)

   >>> info

5. Exit the CLI:

   >>> exit

=== Service Endpoints (ClusterIP) ===

SMTP: {{ include "proton-bridge.fullname" . }}:{{ .Values.service.smtp }}
IMAP: {{ include "proton-bridge.fullname" . }}:{{ .Values.service.imap }}

To access from outside the cluster, use port-forward:

   kubectl --namespace {{ .Release.Namespace }} port-forward $POD_NAME \
     {{ .Values.service.smtp }}:{{ .Values.service.smtp }} \
     {{ .Values.service.imap }}:{{ .Values.service.imap }}
