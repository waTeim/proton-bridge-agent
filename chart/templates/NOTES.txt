Proton Mail Bridge deployed.

The keychain initContainer runs automatically on first boot and sets up the GPG
key and pass store needed by the bridge. It is skipped on subsequent restarts.

=== One-time Proton account login ===

Wait for the pod to reach Running state:

  kubectl get pod -n {{ .Release.Namespace }} \
    -l app.kubernetes.io/instance={{ .Release.Name }} -w

{{- if .Values.sidecar.enabled }}
Log in via the bridge-ctl CLI bundled in the sidecar container:

  kubectl exec -it -n {{ .Release.Namespace }} \
    {{ include "proton-bridge.fullname" . }}-0 \
    -c bridge-sidecar -- bridge-ctl
  # Choose: 1) Login

After login the IMAP watcher starts automatically. On all subsequent pod restarts
the sidecar restores the session from the vault — no manual re-login required.

Sidecar REST API (port {{ .Values.sidecar.port }}):
  GET  /api/v1/credentials/status   — idle / pending / connected / error
  GET  /api/v1/credentials          — IMAP username + bridge password
  POST /api/v1/credentials          — start async login
  PUT  /api/v1/credentials          — logout + re-login
  DELETE /api/v1/credentials        — logout
  GET  /swagger/index.html          — Swagger UI
{{- else }}
The sidecar is disabled (sidecar.enabled=false). To enable the bridge-ctl login
CLI and automatic session restore on restart, redeploy with:

  --set sidecar.enabled=true
  --set sidecar.image.repository=<your-registry>/proton-bridge-sidecar
  --set sidecar.image.tag=<tag>
{{- end }}

=== Service Endpoints (ClusterIP) ===

  SMTP: {{ include "proton-bridge.fullname" . }}:{{ .Values.service.smtpPort }}
  IMAP: {{ include "proton-bridge.fullname" . }}:{{ .Values.service.imapPort }}

Local access via port-forward:

  kubectl port-forward svc/{{ include "proton-bridge.fullname" . }} \
    -n {{ .Release.Namespace }} \
    {{ .Values.service.smtpPort }}:{{ .Values.service.smtpPort }} \
    {{ .Values.service.imapPort }}:{{ .Values.service.imapPort }}
