Proton Mail Bridge deployed.

The keychain initContainer runs automatically on first boot and sets up the GPG
key and pass store needed by the bridge. It is skipped on subsequent restarts.

=== One-time Proton account login ===

Wait for the pod to reach Running state:

  kubectl get pod -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }} -w

Then exec into the pod and log in:

  kubectl exec -it -n {{ .Release.Namespace }} {{ include "proton-bridge.fullname" . }}-0 \
    -- /usr/lib/protonmail/bridge/bridge --cli

Inside the bridge CLI:
  login                       (enter your Proton credentials and follow the 2FA prompts)
  info                        (verify the session is active)
  updates autoupdates disable (answer 'yes'; prevents downloading incompatible binaries)
  exit

Credentials and settings are persisted in the PVC and survive restarts;
this step is only needed once per PVC.

=== Service Endpoints (ClusterIP) ===

  SMTP: {{ include "proton-bridge.fullname" . }}:{{ .Values.service.smtpPort }}
  IMAP: {{ include "proton-bridge.fullname" . }}:{{ .Values.service.imapPort }}

Local access via port-forward:

  kubectl port-forward svc/{{ include "proton-bridge.fullname" . }} \
    -n {{ .Release.Namespace }} \
    {{ .Values.service.smtpPort }}:{{ .Values.service.smtpPort }} \
    {{ .Values.service.imapPort }}:{{ .Values.service.imapPort }}
